# Журнал работ на строительном объекте

Внутренний инструмент для ведения учёта выполненных работ.

## Стек

| Слой | Технология |
|------|-----------|
| Фронтенд | React 19 + TypeScript + Vite |
| UI | shadcn/ui + Tailwind CSS 4 |
| Состояние | Redux Toolkit + RTK Query |
| Формы | react-hook-form + Zod |
| Бэкенд | NestJS 11 + TypeScript |
| ODM | Mongoose 9 |
| База данных | MongoDB 7 |
| Деплой | Docker + docker-compose + nginx |

**Почему этот стек?**
- NestJS — модульная архитектура с DI, идеально для масштабируемых API. Встроенный Swagger экономит время на документацию.
- RTK Query встроен в Redux Toolkit — не нужно держать два отдельных инструмента для кэширования серверных данных и управления состоянием.
- shadcn/ui — компоненты копируются в проект, нет зависимости от внешней библиотеки, полная кастомизация.
- MongoDB + Mongoose — гибкая схема для строительных данных, не требует миграций при изменении структуры.

## Быстрый старт

### Docker (рекомендуется)

```bash
docker compose up -d
```

Открыть http://localhost  
Swagger API: http://localhost:3000/docs
